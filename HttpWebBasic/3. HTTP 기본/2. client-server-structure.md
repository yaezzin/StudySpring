# # 3-2. 클라이언트 서버 구조

## 클라이언트 서버 구조

<img width="621" alt="스크린샷 2022-03-21 오후 2 54 37" src="https://user-images.githubusercontent.com/97823928/159211385-f85c2eae-cdf4-469b-a31a-b8c3f6d4cf2a.png">

* 클라이언트는 서버에 요청을 보내고, 응답을 대기
* 서버가 요청에 대한 결과를 만들어서 응답
* 클라이언트와 서버를 개념적으로 분리해냈기 때문에 **서버는 비지니스 로직과 데이터를 담당하고, 클라이언트는 UI를 그리는데에 집중**
* 클라이언트와 서버가 각각 독립적으로 진화 가능
